Performance

  • Latest angular features(Built-in control flow, defer, trackBy, etc)
  • ChangeDetectionStrategy.OnPush + Signals + Observables
  • Use ESlint rules
  • Use Typescript strict mode
  • Clear folder structure, clean and dry code - use interfaces, ES6, etc.
  • AOT Compilation
  • Angular router for lazy routing
  • Ngrx state management
  • Reactive programming
  • Reactive form
  • Standalone component-based MVC
  • PWA Service Worker
  • NgOptimizedImage - preconnect, lazy load images, eager load images as needed
  • Images in Webp format
  • Prevent memory leaks - use async pipe, libs or rxjs operators to unsubsribe observables
  • No other subscriptions inside a subscription
  • Use SSR or SSG(for webapps like dashboard, this is not necessary)
  • Use gzip compression
  • Documentation - *storybook*

Marketing

  • SEO - Title, Meta(description, canonical link, open graph)
  • Schema Markup - enable rich snippet and content
  • Server Side rendering SSR or Static Site Generation SSG using Angular Universal with DOM hydration- faster initial load time(FCP, LCP, etc) than CSR
  • Semantic html - header, footer, article, section, anchor, button
  • Structured header tags - h1, h2, h3, h4
  • Path Location Strategy - HTML5 routing
  • Images - lazy-loaded, alt text, webp
  • Submit xml sitemap - for faster crawling and indexing process
  • Disallow irrelevant pages in Robots.txt
  • Accessibility
  • Google Tag Manager, Google Analytics, Big Query and Looker Studio

Security

  • Use latest stable Angular and dependencies
  • AOT Compilation to prevent XSS attack(Script injection)
  • When dynamically rendering HTML content, use binding innerHTML(built-in sanitized), to prevent XSS
  • As necessary, use DomSanitizer's sanitze method or library like Dompurify
  • Use DomSanitizer's bypassSecurityTrustHtml only on trusted and/or sanitized HTML content
  • No direct DOM manipulation - e.g. ElementRef's nativeElement or Document's getElementById, use Renderer2 instead for safety and Angular stability(not bypassing change detection, lifecycle hooks) reasons
  • Global(API or client) errors handling
  • Route Guard
  • HTTPS, OAuth2, JWT Authentication
  • Use environment variables - keep api keys and secrets safe
  • Stay educated

Testing

  • Unit testing - Karma and Jasmin
  • E2E testing - Cypress
  • Set up staging and production environments for testing
  • Use sandbox on staging system for payment dependencies like Paypal, Klarna
  • Lighthouse
  • *Angular DevTools*
